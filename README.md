## 🎯 What is InterviewGhost?

InterviewGhost is a full-stack AI mock interview platform that simulates real technical and behavioral interviews. It generates unique questions tailored to your role and difficulty level, evaluates your answers across multiple dimensions, and gives you instant detailed feedback — just like a real interviewer would.

**Built by a sophomore CS student who wanted to stand out in internship recruiting.** No recycled question banks. No vague feedback. Just real interview simulation powered by Claude AI.

---

## ✨ Features

| Feature | Description |
|---|---|
| 🤖 **Claude AI Questions** | Every question uniquely generated by Anthropic's Claude — never the same interview twice |
| 📊 **Multi-Dimensional Scoring** | Scored on Correctness, Clarity, Relevance, and Communication |
| 🎯 **Role-Specific** | 8 roles supported — SWE, Frontend, Backend, Full Stack, Data Science, PM, DevOps, UX |
| 🎤 **Voice Input** | Speak your answers out loud via Web Speech API — just like a real interview |
| ⚡ **Instant Feedback** | Strengths, improvements, ideal answer, and next steps in seconds |
| 📈 **Session History** | Every interview saved — track your progress over time |
| 🔐 **JWT Auth** | Secure registration and login with hashed passwords |

## 🛠 Tech Stack

### Frontend
- **React** + **Vite** — fast, modern frontend tooling
- **React Router** — client-side routing
- **Web Speech API** — voice input for answers
- Custom dark cyberpunk UI — glassmorphism, animated orbs, SVG score rings, confetti

### Backend
- **Node.js** + **Express** — RESTful API
- **JWT** — stateless authentication
- **bcrypt** — password hashing
- **Anthropic Claude API** — AI question generation + answer evaluation

### Database
- **PostgreSQL** on **Neon** — serverless Postgres
- **pg** (node-postgres) — database client

### Deployment
- **Frontend** → Vercel
- **Backend** → Render
- **Database** → Neon

## 🔄 How It Works

```
1. User selects role + difficulty
          ↓
2. Claude AI generates 5 tailored interview questions
          ↓
3. User answers each question (text or voice)
          ↓
4. Claude AI evaluates each answer:
   → Correctness score (0-100)
   → Clarity score (0-100)
   → Relevance score (0-100)
   → Communication score (0-100)
   → Strengths, improvements, ideal answer, next steps
          ↓
5. Overall session score + success probability
          ↓
6. Session saved to database for progress tracking
```

## 🚀 Getting Started

### Prerequisites
- Node.js 18+
- PostgreSQL database (or Neon free tier)
- Anthropic API key

### 1. Clone the repo
```bash
git clone https://github.com/KakadiyaSmit/interviewghost.git
cd interviewghost
```

### 2. Set up the backend
```bash
cd server
npm install
```

Create `server/.env`:
```env
PORT=5000
DATABASE_URL=your_postgresql_connection_string
JWT_SECRET=your_jwt_secret
ANTHROPIC_API_KEY=your_anthropic_api_key
```

### 3. Set up the database

Run this SQL on your PostgreSQL database:

```sql
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  email VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  name VARCHAR(255),
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE sessions (
  id SERIAL PRIMARY KEY,
  user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
  role VARCHAR(100),
  difficulty VARCHAR(50),
  status VARCHAR(50) DEFAULT 'in_progress',
  overall_score INTEGER,
  success_probability INTEGER,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
  completed_at TIMESTAMP
);

CREATE TABLE questions (
  id SERIAL PRIMARY KEY,
  session_id INTEGER REFERENCES sessions(id) ON DELETE CASCADE,
  question_text TEXT,
  question_type VARCHAR(50),
  order_num INTEGER,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE answers (
  id SERIAL PRIMARY KEY,
  question_id INTEGER REFERENCES questions(id) ON DELETE CASCADE,
  user_id INTEGER REFERENCES users(id) ON DELETE CASCADE,
  answer_text TEXT,
  correctness_score INTEGER,
  clarity_score INTEGER,
  relevance_score INTEGER,
  communication_score INTEGER,
  overall_score INTEGER,
  strengths TEXT,
  improvements TEXT,
  next_steps TEXT,
  ideal_answer TEXT,
  created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4. Set up the frontend
```bash
cd ../client
npm install
```

Create `client/.env`:
```env
VITE_API_URL=http://localhost:5000
```

### 5. Run locally
```bash
# Terminal 1 — backend
cd server && node index.js

# Terminal 2 — frontend
cd client && npm run dev
```

Open [http://localhost:5173](http://localhost:5173) 🎉
